Built for multi-camera shoots

Every camera has its own quirks. Jetlag knows them all.

Camera-aware timestamp fixing

Understands how GoPro, iPhone, and Insta360 each store time. Corrects timezone mismatches, FAT filesystem quirks, and missing timezone info.

Final Cut Pro native

Sets Keys:CreationDate and file birth time so FCP's "Content Created" field is always correct — no manual sorting, no timecode drift.

Profile-based workflow

Define camera profiles once in media-profiles.yaml. Run the same pipeline shoot after shoot with one command or one click.

Gyroflow integration

Automatically generates .gyroflow project files for every clip with gyro data, ready for the Gyroflow Toolbox FCP plugin.

SwiftUI native app

A clean macOS menu-bar companion: pick a profile, set a timezone, hit Run. No Terminal required — but the scripts work standalone too.

Auto-organize by date

Moves files into YYYY/Group/YYYY-MM-DD folders after fixing. Alphabetical, interrupt-safe, idempotent.

How it works

Four stages, one pipeline. Run them together or individually.

1

Tag & identify

Each file is matched to a camera profile by EXIF Make/Model. The right fix strategy — GoPro FAT quirks, iPhone timezone string, Insta360 filename parsing — is applied per-camera.

2

Fix timestamps

DateTimeOriginal is the source of truth (never modified). UTC fields, Keys:CreationDate, and file birth time are all corrected to agree — one exiftool read + one write per file.

3

Organize into folders

Files are moved into YYYY/Group/YYYY-MM-DD directories. Empty folders are cleaned up. If the run is interrupted, it resumes safely from where it stopped.

4

Import into Final Cut Pro

Drag your folder into FCP. "Content Created" is set correctly for every clip. Multicam clips, smart collections, and timeline sorting all work with zero manual intervention.
